# Workflow: Google Sheets Integration

## Objective
Read from and write to Google Sheets as the primary output destination for workflow deliverables.

## Prerequisites
- `credentials.json` in project root (OAuth 2.0 Desktop client, downloaded from Google Cloud Console)
- Google Sheets API and Google Drive API both enabled on the project
- `token.json` generated by running the auth tool (see Setup below)

## Setup (one-time)
```bash
source .venv/bin/activate
python tools/google_auth.py
```
This opens a browser window. Log in and grant access. `token.json` is saved automatically and refreshes silently on subsequent runs.

## Finding a Sheet ID
The Sheet ID is in the URL:
```
https://docs.google.com/spreadsheets/d/<SHEET_ID>/edit
```

## Tools

### Read data from a sheet
```bash
python tools/sheets_read.py \
  --sheet-id <SHEET_ID> \
  --range "Sheet1!A1:Z100"
```
- Outputs a JSON 2D array to stdout
- Use `--output .tmp/data.json` to write to a file instead

### Write data to a sheet
```bash
python tools/sheets_write.py \
  --sheet-id <SHEET_ID> \
  --range "Sheet1!A1" \
  --input .tmp/data.json
```
- `--mode overwrite` (default): replaces existing content in the range
- `--mode append`: adds rows below existing data
- Use `--values '[["col1","col2"],["val1","val2"]]'` for inline data

## Typical Pattern
1. Process data → write 2D array to `.tmp/output.json`
2. Run `sheets_write.py` to push to the target sheet
3. Share the sheet URL with the user

## Edge Cases & Notes
- Token expires after ~1 hour of inactivity but auto-refreshes as long as `token.json` exists
- If the OAuth consent screen shows "unverified app", click "Advanced" → "Go to app" — this is normal for personal projects
- Range must include the sheet tab name (e.g. `Sheet1!A1`) if the spreadsheet has multiple tabs
- `USER_ENTERED` input mode means formulas and date strings are interpreted by Sheets, not stored as raw strings
